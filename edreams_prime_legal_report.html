1. Statement of Findings

This document serves as formal technical proof of unauthorized consumer enrollment into the eDreams Prime recurring subscription program. Forensic extraction of the eDreams web application state (sessionStorage payload) immediately prior to checkout proves that eDreams systematically ignores explicit user opt-outs ("Not interested" button clicks) and forcefully enrolls consumers via obscured shopping cart state manipulations.

2. Forensic Evidence (Session Payload)

The following JSON payload was intercepted from the client's browser DOM strictly during the final payment submission phase. The data incontrovertibly shows the client explicitly rejecting the membership, yet the cart forcing the enrollment tag.

// EXCERPT FROM EDREAMS CHECKOUT PAYLOAD "isPrimeSelectedInMembershipPage": false, "flightSelectionData": { "priceWithoutDiscountSelected": false, "extensions": [{ "name": "ESTIMATED_COLLECTION_METHOD", "value": "MEMBER_PRICE_POLICY_DISCOUNTED" }] }

Mechanism of Fraudulent Enrollment

The forensic data reveals a coordinated "Bait-and-Switch" Dark Pattern:

  1. The client initially interacts with aggressively highlighted UI elements promoting a "Prime Price". The system silently attaches the MEMBER_PRICE_POLICY_DISCOUNTED tag to the backend payload.
  2. When subsequently presented with the Prime Subscription enrollment modal, the client explicitly clicked "Not interested". The client's browser correctly logged this explicit refusal (isPrimeSelectedInMembershipPage: false).
  3. The Violation: Clicking the refusal button merely hides the modal visually but intentionally fails to revert the backend cart tag. The eDreams payment processor ignores the client's explicit refusal (the false boolean) and authorizes the subscription strictly based on the hidden ESTIMATED_COLLECTION_METHOD tag.

3. ISO/IEC 27001 Compliance Violations

eDreams purports to be an ISO/IEC 27001 certified organization. The practices documented in this exhibit represent severe breaches of ISO 27001 Information Security Management standards, specifically regarding Data Integrity and Non-Repudiation:

A.8.2.3 / A.14.1.1 (Information Security Requirements): ISO 27001 mandates that applications must maintain the integrity of user input. By ignoring the explicit user boolean `isPrimeSelectedInMembershipPage: false` and instead relying on a hidden cart variable to process financial transactions, eDreams is manipulating user consent data and failing to maintain input integrity.
A.18.1.1 (Identification of Applicable Legislation): Forging consent for recurring financial charges violates European PSD2 (Payment Services Directive) and GDPR standards regarding explicit, unambiguous consent, directly jeopardizing their compliance audit status.

4. Class Action & Regulatory Implications

Legal Basis for Complaint: This payload evidence supports claims of Unfair Commercial Practices Directive (UCPD) violations within the EU, and Wire Fraud / Deceptive Trade Practices in the US. The Italian Competition and Market Authority (AGCM) and the European Consumer Centers Network (ECC-Net) have actively investigated eDreams for these exact deceptive UI practices. This forensic report demonstrates that the issue is not user error, but hardcoded system deception.
